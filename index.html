Final Clock 1 — Meeting Agenda Clock

Sketch iteration documentation (paper/hand-drawn photos)

Design process: The Meeting Agenda Clock began as a simple pie chart divided by agenda sections. In an early version, all slices were the same neutral grey — a classmate gave feedback that it was impossible to tell which section was active at a glance. This led to the first key decision: categorical color encoding, assigning each meeting phase (Intro, Discussion, Wrap-up, Q&A) a distinct hue for preattentive identification, meaning users recognize the active section within 200ms without conscious counting. The second iteration added a real-time countdown with a used/remaining shadow split: the dark region grows as time elapses, leveraging the area-ratio perceptual channel, which research shows is effective for proportion judgments. A peer reviewer noted that the shadow effect was too subtle at first — I increased the contrast to fill(60, 60, 60, 230) so the consumed portion is immediately obvious even in peripheral vision. Finally, I added click-to-start interaction and hover highlights, applying the principle of direct manipulation: users act on the visual object itself rather than a separate control, reducing cognitive load in a live meeting context.

Self-reflection / future work: First, I would add audio cues or a subtle flash when a section's time runs out, since the current "TIME UP" text is easy to miss if the user is not looking at the screen. Second, I would allow users to customize the number and duration of agenda sections, making the clock adaptable to different meeting formats beyond the fixed 4-section layout.

Final Clock 2 — Work Clock

Sketch iteration documentation (paper/hand-drawn photos)

Design process: The Work Clock went through three major structural iterations, each driven by feedback. The first version used rotating gears (sketch 5) — a friend who tested it said the spinning motion made the clock hands unreadable and the design felt more decorative than functional. This feedback directly motivated the pivot to a cleaner circular layout. The second iteration used three fixed clocks with shrinking arcs, but a second reviewer noted there was no way to know which task was most urgent — all three clocks looked equally important. This led to the third key decision: size encodes priority. Clicking a clock cycles it through HIGH / MID / LOW states, changing its radius so the most urgent task visually dominates the layout. This applies the size preattentive channel — larger objects draw attention first without requiring conscious comparison. Color redundantly reinforces the same hierarchy: red for HIGH urgency, amber for MID, blue for LOW, exploiting the culturally universal red-danger / blue-calm association. The smooth rotation driven by frameCount rather than second() was a technical decision made after observing that second()-based motion created jarring one-second jumps — frameCount produces continuous, fluid motion that feels alive and responsive, consistent with the goal of making the visualization feel dynamic.

Self-reflection / future work: First, I would add the ability to reset or customize deadlines via a simple input field, since the current deadlines are hardcoded at page load. Second, I would explore adding a subtle sound or browser notification when a task hits the low priority threshold, because visual changes alone can be missed if the user is focused on another window.

Context of use: This sketch represents the rhythm of box breathing, a meditation technique used to calm the mind and reduce stress by maintaining a steady 4-4-4-4 breathing cycle.

Ex 11

Design process:

  1. Square path visualization: The breathing cycle is represented by a square with four edges, where each edge corresponds to one breathing phase.
  2. Color-coded phases: Each of the four phases uses a distinct teal-to-gray gradient color.
  3. Countdown timer with neon glow: Large text displays the current phase label with a calming neon glow effect.

Self-reflection / future work: Add customizable breathing durations and audio cues for eyes-closed practice.
